# HTTP 状态码速查

## 五大类

- **1xx 信息**：继续处理，如 100 Continue、101 Switching Protocols
- **2xx 成功**：200 OK、201 Created、204 No Content
- **3xx 重定向**：301 永久重定向（搜索引擎更新）、302 临时重定向、303 See Other、304 Not Modified（协商缓存命中）、307/308 不允许更改方法
- **4xx 客户端错误**：400 Bad Request、401 未认证、403 无权限、404 未找到、405 方法不允许、413 请求体过大、429 Too Many Requests
- **5xx 服务端错误**：500 内部错误、502 网关错误、503 服务不可用（可带 `Retry-After`）、504 网关超时

## 缓存相关

- **200 (from cache)**：强缓存命中，直接返回本地缓存。
- **304 Not Modified**：协商缓存命中，服务端确认资源未变更。
- 相关头：`Cache-Control`、`ETag`/`If-None-Match`、`Last-Modified`/`If-Modified-Since`。

## 重定向区别

- **301**：资源永久迁移，浏览器会缓存，搜索引擎更新索引。
- **302/307**：临时重定向，方法不变；302 历史遗留可能改方法，307 明确不改。
- **303**：重定向为 GET，常用于表单提交后跳转。

## 排查思路

- 4xx 优先检查请求：URL、方法、参数、鉴权头。
- 5xx 查看服务日志、依赖链路；503/504 多为上游不可用或超时。
- 3xx 循环重定向：检查 Location 配置、协议/域名混用。
