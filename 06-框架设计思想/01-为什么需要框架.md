# 为什么会出现 React / Vue 这些框架？

## 1. 历史演变 (Pain Points)

### 时代 1：原生 JS + HTML (刀耕火种)
- **问题**：代码混乱，JS 里拼 HTML 字符串，难以维护。
- **痛点**：缺乏模块化。

### 时代 2：jQuery (工具库时代)
- **问题**：**频繁手动操作 DOM**。
- **痛点**：
  - 数据变了，要手动找 DOM 改内容 (`$('.box').text(data)`)。
  - 页面复杂时，很难理清数据和视图的对应关系（"面条代码"）。
  - 性能瓶颈：频繁回流重绘。

### 时代 3：AngularJS / React / Vue (MVVM 框架时代)
- **变革核心**：**数据驱动视图 (Data-Driven)**。

---

## 2. 框架解决的核心问题

### 1. 声明式编程 vs 命令式编程

- **命令式 (jQuery)**：关注"怎么做"。
  > "找到 id 为 app 的 div，清空它，创建一个 span，设置内容为 hello，插入 div。"
- **声明式 (Vue)**：关注"结果是什么"。
  > "这个 div 的内容就是 `message` 变量。"
  
  **价值**：降低心智负担，开发者只需关心数据，无需关心 DOM 操作细节。

### 2. 组件化 (Component)

- 将 UI 拆分为独立、可复用的积木。
- **高内聚**：HTML/CSS/JS 都在一个组件里。
- **低耦合**：组件间通过标准接口 (Props/Events) 通信。
- **价值**：极大提升了大型项目的可维护性和开发效率。

### 3. 性能优化 (Virtual DOM)

- 框架内部实现了**批量更新**和**Diff 算法**。
- 保证了性能下限，避免新手写出导致页面卡死的垃圾 DOM 操作代码。

### 4. 跨平台能力

- 虚拟 DOM 使得一次编写，多端运行（Web / iOS / Android / 小程序）成为可能。

---

## 3. React 和 Vue 的设计哲学差异

| 维度 | React | Vue |
|---|---|---|
| **核心思想** | **JS 为中心** (All in JS) | **HTML 为中心** (模板语法) |
| **数据流** | 单向数据流 | 单向数据流 (但提供 v-model 双向糖) |
| **响应式** | 手动触发 (`setState`) | 自动追踪 (`Proxy` / `setter`) |
| **自由度** | 极高 (由于 JSX) | 适中 (有明确的规范和 API) |
| **学习曲线** | 陡峭 (需理解 FP, Hooks) | 平缓 (易上手) |

## 总结

框架的出现不是为了让简单的网页变复杂，而是为了让**构建复杂的大型应用变得可控和简单**。它们通过**数据驱动**解救了陷于 DOM 操作泥潭的开发者。
